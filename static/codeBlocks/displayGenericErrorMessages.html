While it should never be considered as an effective countermeasure in itself, displaying generic error messages can increase the level of sophistication required to exploit a programming flaw, because the attacker must conduct the exploit blindly, without relying on the application output.


For example an SQL Injection attack may be a bit harder to detect and to exploit if the application only displays the message: 

An error has ocurred


Obviously if the application displays the message below, it is much easier to exploit because the attacker has the exact query and does not need to do any guessing. 

java.sql.SQLSyntaxErrorException: Syntax error: Encountered "\'" at line 1, column 49.
Query:select * from users where usr='admin' and pwd='''


Displaying detailed error messages is a vulnerability in itself although not one of the top Sans 25 or OWASP Top 10. Many times such messages may reveal information about the system being attacked such as file paths or exception call stacks. These errors make the attacker's life much easier.
